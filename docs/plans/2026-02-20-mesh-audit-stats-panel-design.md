# Mesh Audit Stats Panel Design

**Date:** 2026-02-20  
**Status:** Approved  
**Context:** Adding a collapsible stats info panel to the PulseMeshAudit node UI to display mesh audit metrics alongside the 16-image carousel.

---

## Overview

Enhance the mesh audit node UI with an "Asset Stats" collapsible accordion panel displaying mesh statistics parsed from the `audit_log.json` generated by agnirt. The panel displays three categories of information: Scene Metadata, Geometry Stats, and Quality Metrics.

---

## Layout & Presentation

### Node UI Structure
```
┌─────────────────────────────────────────────────┐
│                                                 │
│  Carousel (Left)        │  Asset Stats (Right)  │
│  16-image grid          │  Collapsible Panel    │
│                         │                       │
│                         │  ▼ Asset Stats        │
│                         │  ├─ Scene Metadata    │
│                         │  ├─ Geometry Stats    │
│                         │  └─ Quality Metrics   │
│                         │                       │
└─────────────────────────────────────────────────┘
```

### Accordion Behavior
- **Header**: "Asset Stats" — clickable to expand/collapse
- **Collapsed state**: Only header visible
- **Expanded state**: Three sections visible below header:
  1. Scene Metadata
  2. Geometry Stats
  3. Quality Metrics

---

## Data Structure

### Scene Metadata
```
Asset Name: <asset_name>
Timestamp: <YYYY-MM-DD HH:MM:SS>
```

### Geometry Stats
```
Edge Count: <number>
Face Count: <number>
Vertex Count: <number>
```

### Quality Metrics
```
Degenerate Triangles: <percentage>%
Sliver Triangles: <percentage>%
Inverted Triangles: <percentage>%
```

All numeric values displayed as plain text (no progress bars, color coding, or badges at this stage).

---

## Data Flow

### Backend (mesh_audit_node.py)
1. After agnirt execution, locate `audit_log.json` in the same temp directory as output images
2. Parse JSON and extract:
   - `asset.name`, `asset.timestamp`
   - `scene_stats.edge_count`, `scene_stats.face_count`, `scene_stats.vertex_count`
   - `validation.degenerate_triangles` (use count to calculate %)
   - `validation.sliver_triangles.percentage`
   - `validation.inverted_triangles` (use count to calculate %)
3. Return structured data in node response under new key: `"asset_stats"`

Example response:
```python
return {
    "ui": {
        "images": [...],
        "mesh_audit_carousel": [{...}],
        "asset_stats": {
            "scene": {
                "name": "BrainStem",
                "timestamp": "2026-02-20 12:25:14"
            },
            "geometry": {
                "edge_count": 39875,
                "face_count": 20066,
                "vertex_count": 21178
            },
            "quality_metrics": {
                "degenerate_triangles_pct": 0.15,
                "sliver_triangles_pct": 8.34,
                "inverted_triangles_pct": 27.82
            }
        }
    }
}
```

### Frontend (mesh_audit_carousel.js)
1. Receive `asset_stats` from node response
2. Build accordion DOM structure with three sections
3. Populate text content with parsed values
4. Attach click handler to "Asset Stats" header to toggle `expanded` class
5. Use CSS to show/hide content sections based on state

---

## Implementation Notes

- **Error handling**: If `audit_log.json` is missing or unparseable, gracefully skip stats (log warning, don't crash)
- **Percentage calculation**: Some values in audit_log are already percentages (sliver_triangles.percentage); others may need calculation from count/total
- **CSS styling**: Use ComfyUI's existing widget styles; minimal new CSS needed
- **No breaking changes**: All changes are additive; existing carousel functionality unchanged
